# Data Science & Version Control Reference Guide 🛠️

This document outlines core digital literacy skills, essential command-line workflows, and tools required for reproducible data science projects.

## 1. Version Control Essentials (Git)

These commands help track project changes systematically:

* `git init` - Initialize a new Git repository.
* `git status` - Check the current status of files.
* `git add <filename>` - Stage files for commit.
* `git commit -m "message"` - Save changes to repository history.
* `git push origin main` - Upload local commits to GitHub.

## 2. Core Data Science Stack

* **Development Environments:** Jupyter Notebook, VS Code
* **Programming:** Python, Pandas, NumPy
* **Data Sources:** CSV files, SQL databases, APIs

## 3. Data Literacy Best Practices

* **Reproducibility:** Document project dependencies using `requirements.txt`.
* **Version Control:** Commit changes regularly with meaningful commit messages.
* **Data Privacy:** Use `.gitignore` to exclude sensitive files and credentials from public repositories.

---

### Key Takeaway

Digital literacy in data science involves not only writing code but also managing projects efficiently, maintaining reproducibility, and protecting sensitive information.
